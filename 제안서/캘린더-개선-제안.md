# 📅 캘린더 개선 제안서

> 작성일: 2026-04-14
> 대상: my-dashboard v2

## 개요

현재 캘린더 페이지는 기본 기능(월간 뷰, 일정 CRUD, 태그, 날씨, 여행 탭 통합, 공유)이 완성돼 있음. 이 문서는 시니어 기획자 관점에서 **더 쓸모 있고 세련된** 앱으로 만들기 위한 개선 제안 15가지를 담고 있음.

## 범례

- 🔴 **미구현**
- 🟢 **완료**
- ⭐ 중요도
- ⚡ 작업량 (적음 → 많음)

---

## ⭐⭐⭐ 우선순위 상

### 1. 대시보드 홈 위젯 🔴

**문제**: 현재 `/`는 `/calendar`로 redirect만 함. 앱을 여는 순간의 "첫인상"이 낭비됨.

**제안**: `/`를 대시보드 홈으로 만들어서 한눈에 전체 상태를 파악:
- 오늘의 일정 목록 (최대 5개)
- 오늘/이번주 지출 합계 + 여윳돈
- 복용/사용 중인 생필품 리마인드
- 최근 가본 날 타임라인 (가장 최근 3개)
- 미읽음 알림 요약
- 공유된 사용자 아바타 스택

**기대 효과**: 앱을 여는 순간 "오늘 뭐해야 하지"가 즉시 해결. 슈퍼앱 느낌.

**작업량**: ⚡⚡ (이미 있는 데이터 조합)

---

### 2. 고정비 결제일 → 캘린더 자동 이벤트 🔴

**문제**: 고정비(예: 매월 11일 카드값)가 `expenses`와 `fixed_expenses`에만 쌓이고 **캘린더에는 안 보임**. "언제 돈 나가지?"가 시각화되지 않음.

**제안**:
- `fixed_expenses`의 각 항목에 대해 매월 해당 일자에 **읽기 전용 캘린더 이벤트** 자동 생성/표시 (🏷 아이콘)
- DB에 저장하지 않고 캘린더 조회 시점에 가상으로 합치기 (읽기 전용이라 OK)
- 클릭하면 가계부의 해당 고정비로 이동

**기대 효과**: 돈 나가는 날이 시각적으로 체감됨.

**작업량**: ⚡⚡

---

### 3. 자연어 일정 입력 🔴

**문제**: 모바일에서 날짜/시간 일일이 클릭해서 입력하기 귀찮음.

**제안**: `chrono-node` 라이브러리로 자연어 파싱:
- "내일 3시 카페" → 내일 15:00 제목=카페
- "이번주 금요일 저녁 미팅" → 금요일 18:00 제목=미팅
- "다음달 7일 약속" → 2026-05-07
- 새 일정 폼에 **빠른 입력** 인풋 추가 → 파싱 결과 프리뷰 → 저장

**기대 효과**: 모바일 입력 속도 3배↑

**작업량**: ⚡⚡

---

### 4. 반복 일정 시리즈 관리 🔴

**문제**: 반복 일정 삭제/수정 시 **"이 일정만 / 이후 모두 / 전부"** 옵션이 없음. 하나만 바꾸고 싶어도 전체가 망가짐.

**제안**:
- 반복 일정 수정 시 확인 팝업:
  ```
  이 반복 일정을 어떻게 수정할까요?
  ( ) 이 일정만
  ( ) 이후 모든 일정
  ( ) 모든 반복 일정
  ```
- DB 스키마: `series_id` 컬럼 추가해서 같은 시리즈 묶기

**기대 효과**: 반복 일정 신뢰도 크게 향상. Google Calendar 수준 UX.

**작업량**: ⚡⚡⚡ (시리즈 모델링 필요)

## ⭐⭐ 우선순위 중

### 5. 날씨 캐시 v2 🟢 **완료**

localStorage에 날짜별 저장, 과거 날씨 영구 / 예보 6시간 TTL.

---

### 6. 이벤트를 다른 사용자에게 드래그로 이전 🔴

**문제**: 공유 대상 사용자에게 "이거 너도 해" 넘기려면 직접 수정해서 user_id 바꿔야 함.

**제안**:
- 캘린더 상단 사용자 칩에 이벤트를 드래그 드롭
- 드롭 시 해당 이벤트의 `user_id`가 대상 사용자로 변경됨 (권한 확인 후)
- 알림 발송: "A가 일정을 보냈어요"

**기대 효과**: 가족/커플 간 "할 일 넘기기" UX. 재미있는 상호작용.

**작업량**: ⚡⚡

---

### 7. 캘린더 공유 수락/거절 🟢 **완료**

`calendar_shares` 테이블 + 알림 패널 수락/거절 버튼.

---

### 8. 일정 템플릿 🔴

**문제**: "매주 운동", "매일 아침 루틴" 같은 자주 쓰는 일정을 매번 새로 만들어야 함.

**제안**:
- 임의의 이벤트를 우클릭 → **"템플릿으로 저장"**
- 템플릿 저장소 (로컬 or DB `event_templates`)
- 새 일정 폼에 **템플릿에서 선택** 옵션 → 한 번 클릭으로 제목/시간/색상/태그 자동 입력

**기대 효과**: 루틴 생성 속도 향상. 데이트 코스 재활용 가능.

**작업량**: ⚡⚡

**작업량**: ⚡ (5분)

### 10. PWA 오프라인 캐시 🔴

**제안**: Service Worker로 지식창고/메모를 오프라인에서도 읽기 가능하게. 네트워크 끊긴 지하철에서도 노트 볼 수 있음.

**작업량**: ⚡⚡⚡ (테스트 복잡)

**제안**: 내 일정을 `.ics` 파일로 내보내기 → Google Calendar, Apple Calendar 등 외부 앱에 import 가능.

**작업량**: ⚡⚡

---

### 12. 이미지 업로드 (데이트 커버) 🔴

**제안**: 여행/데이트 항목에 대표 이미지 업로드. Supabase Storage 무료 1GB.

**참고**: 데이트 필드(분위기/가격대/별점/함께한 메모)는 사용자 요청으로 이미 삭제됨. 필요 시 재검토.

**작업량**: ⚡⚡

---

### 13. D-day 배너 🔴

**제안**: 기념일 일정에 "D-day 기능" 플래그 → 캘린더 상단에 "D-7 일주년" 자동 표시.

**작업량**: ⚡

---

### 14. 월간/주간/일간 뷰 전환 🔴

**제안**: 현재는 월간 뷰만. 주간(7일 타임블록), 일간(하루 시간대별) 뷰 추가.

**작업량**: ⚡⚡⚡ (주간·일간은 새 그리드)

---

### 15. 검색 🔴

**제안**: 상단에 검색 입력 → 제목/태그/설명 전역 검색 → 결과 클릭 시 해당 월로 이동.

**작업량**: ⚡⚡
